# 桌面端布局决策

## 选定方案：主题优先的一屏工具

主窗口按照实际任务顺序组织：顶部显示扫描摘要，中间选择主题，每个客户端卡片直接提供
“正常启动”和“注入主题并启动”。界面不保留目标勾选、所选目标汇总或批量操作；活动与诊断
压缩在右侧窄栏，只展示当前操作需要的信息。

窗口内容不重复展示应用图标、应用名称或独立品牌工具栏。重新扫描与状态摘要并入“主题与应用”
标题行；应用图标只用于系统启动、Dock、Finder 和短暂加载状态。

Manager 自身提供 Light / Dark 两套应用外观。首次启动跟随 macOS，用户在标题行切换后的选择仅在
本机持久化；该设置只影响 Manager 窗口，不进入 Theme Family，也不覆盖任何宿主客户端外观。

这个布局遵循以下原则：

- **先看结果**：用户进入应用就能看到发现数量、可应用数量和运行数量；
- **卡片动作就是目标**：主题与 Adapter Engine 相互独立，点击哪张客户端卡片的注入按钮，就只
  编译并应用该 Adapter，不需要预先维护“所选目标”；
- **能力决定操作**：按钮由 Adapter Capability 和本机探测结果生成；未注册的客户端不会扫描、
  展示或进入安装资源；
- **主要操作无需滚动**：在默认窗口尺寸内同时呈现主题、客户端状态和每端两个启动按钮；
- **细节按需出现**：版本、诊断和活动信息保持简短，不用大段产品说明挤占主流程。

关闭红色窗口按钮不会直接结束进程，而是让用户选择“最小化到菜单栏”或“退出应用”。顶部菜单栏
图标只保留“打开窗口”和“退出”；这两条退出路径都不隐式执行主题 pause/restore，已经注入的
主题不因 Manager 退出而失效。

没有采用以客户端为导航的多页结构，因为它会把主题选择和客户端状态拆散；也没有采用密集的
主题 × 客户端表格，因为表格难以清楚表达运行状态、能力门禁和失败恢复。界面不保留原型切换器
或不可执行的深度设置入口。
