## CC Theme 0.1.0 — Historical bootstrap

- Added a self-contained Claude Desktop Skin Adapter with exact `1.22209.0`
  fingerprint evidence, while failing runtime apply closed because the official
  client requires an Anthropic-signed CDP authorization token.
- Recorded a successful privacy-safe, session-only neutral external-theme paint proof
  through Claude's diagnostic DevTools entry without treating it as an apply Seam.
- Added a machine-callable Manager diagnostic preview contract with explicit user
  confirmation, official DevTools launch, CSP-safe two-step bootstrap, tokenized
  loopback receipts, structural-only verification and normal-restart cleanup.
- Added localized animated-image fallback playback states so the UI does not
  misreport Claude's GIF media fallback as native video playback.
- Added the Claude-specific UI Surface Catalog, Interpreter Adapter and
  privacy-safe Live Surface Evidence contract.
- Added an independent `CC Theme` Settings tab immediately below General with a
  native-isomorphic entry and a letter C in the native icon container.
- Added a fully WYSIWYG editor with immediate preview, debounced atomic
  latest-write-wins persistence, theme-scoped state and rollback.
- Added machine-readable detect/preflight/apply/verify/pause/restore lifecycle
  results and adapter-landmark/theme-contract/visual failure categories.
- Removed Adapter-owned production authoring and preset distribution in favor of
  the independent CC Theme theme-resource layer.
- Added complete owned DOM/style/listener/Blob/WebGL/Canvas cleanup and retained
  the official Claude application, ASAR and signature as read-only.
- Added automated privacy, settings-entry, WYSIWYG, resource-ownership, lifecycle,
  release and cleanup regression tests.
- Added four-party v1 capability discovery, closed Target Profile, explicit
  exact/approximate/unsupported Projection diagnostics and versionless theme data.
- Added one cross-process Adapter transaction Seam plus compatible override replay
  and deterministic quarantine for invalid values after base-hash changes.
- Made Theme Package identity canonical-only: imports accept exactly one `mac-claude`
  Adapter and reject retired aliases or client-version migration fields.
- Added the Claude-owned 11-locale Catalog, `DesktopIntl` effective-locale runtime,
  complete CC Theme translations, atomic in-place language refresh, bounded fallback,
  localized formatting and listener cleanup without disturbing pending WYSIWYG writes.
